# 처음부터 다시 — 새 저장소 + 시트 복사본으로 v5 배포하기

> 상황: 원본(기존 주간회의 자료)은 **그대로 두고**, 새 GitHub 저장소 + 구글시트 복사본으로
> v5 버전을 따로 띄웁니다. 원본 저장소·원본 시트·원본 사이트는 전혀 건드리지 않습니다.

---

## ⚠️ 먼저, 가장 중요한 것 — 파일 이름

웹사이트가 동작하려면 GitHub 저장소의 파일 이름이 **반드시 아래 영문 그대로**여야 합니다.
지금 한글 이름(`인덱스.html`, `스타일.css`, `앱.js`)으로 올라가 있어서 화면이 안 뜹니다.

| 지금(잘못) | 올바른 이름 | 용도 |
|---|---|---|
| 인덱스.html | **index.html** | 사이트 시작 파일 (필수) |
| 스타일.css | **style.css** | 디자인 (index.html이 이 이름으로 불러옴) |
| 앱.js | **app.js** | 동작 (index.html이 이 이름으로 불러옴) |
| 코드.gs | **Code.gs** | 구글 Apps Script용 (사이트엔 안 쓰임, 참고 보관) |

### 이름 바꾸는 법 (GitHub 웹)
1. 저장소에서 `인덱스.html` 클릭 → 오른쪽 위 **연필(Edit)** 아이콘
2. 화면 맨 위 **파일명 칸**의 `인덱스.html`을 지우고 **`index.html`** 로 입력
3. **Commit changes**
4. 같은 방법으로 `스타일.css → style.css`, `앱.js → app.js`, `코드.gs → Code.gs`

> 더 확실한 방법: 한글 이름 파일들을 모두 삭제하고, 아래에서 다시 받은 파일을
> **영문 이름 그대로** 다시 업로드. (업로드 후 저장소에 `index.html`/`style.css`/`app.js`로 보이는지 꼭 확인)

---

## 1. GitHub 저장소 정리 (새 저장소)

새 저장소(예: `디지털 서비스-새로운-ny`)에 아래 3개 파일이 **저장소 최상단**에 영문 이름으로 있어야 합니다.

```
index.html   ← 사이트 시작 파일
style.css
app.js
```
- `Code.gs`, 가이드 `.md` 파일은 사이트에서 안 쓰입니다. 같이 둬도 무방(참고용).
- 원본 저장소는 그대로 두므로, 이 새 저장소만 작업하면 됩니다.

---

## 2. Vercel에 새 저장소 연결 (독립 테스트 사이트 생성)

새 저장소를 Vercel에 **새 프로젝트로** 가져오면, 원본과 완전히 분리된 테스트 사이트 주소가 생깁니다.

1. https://vercel.com 로그인 → **Add New… > Project**
2. 새 저장소(`디지털 서비스-새로운-ny`) 옆 **Import** 클릭
   - GitHub 저장소 목록에 안 보이면 "Adjust GitHub App Permissions"에서 이 저장소 접근 허용
3. 설정 화면에서:
   - **Framework Preset: Other** (정적 사이트, 빌드 불필요)
   - **Root Directory:** 그대로 `./` (index.html이 최상단에 있으므로)
4. **Deploy** → 잠시 후 완료되면 사이트 주소(예: `프로젝트명.vercel.app`)가 나옵니다. 이게 테스트 주소입니다.

> 이 시점에 사이트를 열면 "데이터를 불러오지 못했습니다"가 나올 수 있습니다 — 정상입니다.
> 아직 백엔드(시트 복사본) 연결 전이라서요. 3·4단계 후 정상화됩니다.

---

## 3. 구글시트 복사본 설정 (백엔드)

복사본 시트(`디지털서비스본부 주간보고 입력 시트 NEW`)에서 진행합니다. **원본 시트는 건드리지 않습니다.**

### 3-1. 시작일/종료일 컬럼 추가
각 주차 시트(`6M-2W` 등)의 `6. 팀별 주요 실적` 표에 컬럼 2개를 맨 뒤에 추가:
```
… I 액션 | J 시작일 | K 종료일
```
- 헤더 행(노출설정 줄)의 J/K에 `시작일`, `종료일` 제목 입력
- 각 업무 행 J/K에 날짜 입력 (`06.01` 형식). **비워두면 화면에 `-` 로 표시** → 일부만 채워도 됨

### 3-2. v5 Code.gs 적용
- 복사본 시트에서 `확장 프로그램 > Apps Script`
- 복사돼 들어온 기존 코드 전체 삭제 → 받은 **`Code.gs`(v5)** 내용 붙여넣기 → 저장

### 3-3. 새 웹앱으로 배포 (새 URL 발급) ★복사본은 URL이 없으므로 필수
- Apps Script에서 **배포 > 새 배포** → 유형 **웹 앱**
- **실행 주체: 나**, **액세스 권한: 모든 사용자** → **배포**
- 권한 승인 화면이 뜨면: "Google에서 확인하지 않은 앱" → **고급 > (프로젝트)로 이동** → 허용
- 완료 후 나오는 **웹앱 URL**(`https://script.google.com/macros/s/…/exec`)을 복사

### 3-4. URL 동작 확인
- 브라우저 주소창에 그 `…/exec` URL 붙여넣기 → JSON 데이터가 나오면 OK

### 3-5. 체크박스 메뉴(선택)
- 시트를 새로고침하면 상단에 `주간보고` 메뉴 생성 → 필요 시 `주간보고 > 노출설정을 체크박스로` 1회 실행

---

## 4. 프론트엔드를 복사본 시트에 연결

새 저장소의 `app.js`가 **복사본 시트의 새 웹앱**을 바라보게 합니다.

1. GitHub 새 저장소에서 `app.js` 클릭 → 연필(Edit)
2. 상단 근처의 이 줄을 찾습니다:
   ```js
   const API_URL = "https://script.google.com/macros/s/…/exec";
   ```
3. 따옴표 안 값을 **3-3에서 복사한 새 웹앱 URL**로 교체 → **Commit changes**
4. Vercel이 자동으로 다시 배포합니다. 2단계의 사이트 주소를 새로고침(Ctrl+F5).

---

## 5. 확인 체크리스트 (Vercel 테스트 주소에서)

- [ ] 화면이 뜨고 "데이터를 불러오지 못했습니다" 에러가 없음
- [ ] 주차 드롭다운 / 새로고침 동작
- [ ] **팀별 주요 실적이 HR 표**(업무·목적·시작일·종료일·진척율·진행사항·지연사유·예정사항)
- [ ] **세로 진척 막대 색상**(100%=그린 / 지연사유 있음=오렌지 / 그 외=블루)
- [ ] 시작일/종료일 입력 행은 날짜, 미입력 행은 `-`
- [ ] 탭 스크롤 / `기존 폼으로 다운로드` 엑셀 / 빈 섹션 자동 숨김

---

## 6. 막힐 때 빠른 점검

| 증상 | 원인/해결 |
|---|---|
| 화면이 아예 안 뜸(빈 화면) | 파일 이름이 `index.html`/`style.css`/`app.js`가 맞는지 (⚠️ 맨 위 참고) |
| 디자인이 깨짐 | `style.css` 이름이 정확한지 |
| "데이터를 불러오지 못했습니다" | `app.js`의 `API_URL`이 새 웹앱 URL인지, 그 URL을 브라우저에서 열면 JSON이 나오는지 |
| "map.get(...).push…" 에러 | 최신 `app.js`(버그 수정본)인지 — 이번에 다시 받은 파일이면 해결됨 |
| 표는 뜨는데 날짜가 다 `-` | 시트 J/K 컬럼에 날짜 입력 여부 (안 넣으면 `-`가 정상) |

---

## 7. 원본은 이렇게 보호됩니다

- 원본 GitHub 저장소: 손대지 않음 (새 저장소에서만 작업)
- 원본 Vercel 사이트: 그대로 운영
- 원본 구글시트 + 원본 웹앱 URL: 그대로 (복사본에서만 새 웹앱 배포)
- 나중에 v5를 정식 적용하고 싶을 때만, 원본에 같은 작업을 반복하면 됩니다.
