# GitHub + Vercel 배포 가이드 — 주간보고 대시보드 v5.0

> 구성: **GitHub**(소스 코드 저장) → **Vercel**(자동 빌드·호스팅)
> 핵심 전략: **새 브랜치**에 v5 파일을 올리면 Vercel이 **별도 미리보기 URL**을 자동 생성합니다.
> → 운영 사이트(production)는 그대로 두고 새 버전을 안전하게 테스트한 뒤, 확인되면 합칩니다.
>
> ※ Apps Script(Code.gs)는 구글 쪽에서 따로 진행합니다(아래 7번). Vercel은 프론트엔드만 호스팅합니다.

---

## 0. 전체 그림

```
[내 PC: 새 파일 3개]
   index.html / style.css / app.js  (v5)
        │  업로드
        ▼
[GitHub]  main 브랜치(운영) ──그대로 유지──▶ Vercel Production (기존 사이트)
          v5 브랜치(신규) ─────────────────▶ Vercel Preview (테스트용 임시 URL)
        │  검증 OK → Pull Request로 main에 병합
        ▼
[Vercel]  Production 자동 재배포 → 정식 반영
```

장점: main(운영)을 건드리지 않으므로 **원본이 사라질 위험 없음**, 잘못돼도 Vercel에서 클릭 한 번으로 이전 배포로 롤백.

---

## 1단계. 사전 확인

1. **GitHub 저장소(레포)** 가 Vercel에 연결돼 있는지 확인
   - Vercel 대시보드(https://vercel.com) 로그인 → 해당 프로젝트 클릭 → **Settings > Git** 에서 연결된 GitHub 저장소 이름 확인.
2. **현재 파일 위치** 확인
   - GitHub 저장소에서 `index.html`, `style.css`, `app.js`가 어느 폴더에 있는지 봅니다. (보통 저장소 최상단(root) 또는 `public/` 같은 폴더)
   - v5 파일도 **같은 위치에 같은 이름으로** 교체할 것입니다. (이 사이트는 빌드가 필요 없는 정적 HTML이라 Vercel이 그대로 서빙합니다.)

---

## 2단계. GitHub에서 새 브랜치 만들기 (웹 브라우저만 사용, 명령어 불필요)

1. https://github.com 로그인 → 해당 저장소로 이동
2. 파일 목록 왼쪽 위의 **브랜치 선택 버튼**(보통 `main` 이라고 적힌 드롭다운) 클릭
3. 입력창에 **`v5`** 라고 입력 → **"Create branch: v5 from main"** 클릭
4. 이제 화면이 `v5` 브랜치로 바뀝니다. (이후 작업은 모두 이 브랜치에서)

> `v5` 브랜치는 main의 복사본입니다. 여기서 무엇을 하든 운영(main)에는 영향이 없습니다.

---

## 3단계. v5 브랜치에 새 파일 3개 올리기

브랜치 표시가 **`v5`** 인지 꼭 확인하고 진행하세요.

### 방법 A — 파일 하나씩 교체 (가장 명확)
각 파일(`index.html`, `style.css`, `app.js`)에 대해 반복:
1. 저장소에서 해당 파일 클릭 → 오른쪽 위 **연필 아이콘(Edit this file)** 클릭
2. 편집창 내용을 **전체 선택(Ctrl+A) 후 삭제** → 새 파일 내용 **붙여넣기**
3. 아래로 스크롤 → **"Commit changes"**
   - 옵션에서 **"Commit directly to the `v5` branch"** 선택되어 있는지 확인 → 커밋

### 방법 B — 한꺼번에 업로드 (드래그 앤 드롭)
1. 저장소(해당 폴더) 화면에서 **"Add file > Upload files"** 클릭
2. 새 `index.html`, `style.css`, `app.js` 3개를 드래그해서 올림 (같은 이름이면 자동 교체됨)
3. 아래에서 **"Commit directly to the `v5` branch"** 선택 → **Commit changes**

---

## 4단계. Vercel 미리보기(Preview) URL 확인

`v5` 브랜치에 커밋하면 Vercel이 **자동으로** 미리보기 배포를 시작합니다.

확인 방법(둘 중 편한 것):
- **Vercel 대시보드** → 프로젝트 → **Deployments** 탭 → 방금 만들어진 `v5` 빌드 클릭 → 상단의 **Visit** 버튼 또는 미리보기 URL(예: `프로젝트명-git-v5-계정명.vercel.app`)
- 또는 GitHub에서 **Pull Request**를 만들면(5단계), PR 화면에 Vercel 봇이 미리보기 URL을 자동 코멘트로 남깁니다.

빌드 상태가 **Ready(초록)** 가 되면 그 URL로 접속해 테스트합니다.

> 이 미리보기 주소는 운영과 완전히 분리된 임시 주소입니다. 마음껏 테스트하세요.

---

## 5단계. 미리보기에서 검증 (체크리스트)

미리보기 URL에서 아래를 확인:
- [ ] 최신 주차 자동 로드 / 주차 드롭다운 / 새로고침 동작
- [ ] **팀별 주요 실적이 HR 표 형식**(업무·목적·시작일·종료일·진척율·진행사항·지연사유·예정사항)
- [ ] **세로 진척 막대 색상**(100%=그린, 지연사유 있음=오렌지, 그 외=블루)
- [ ] 시작일/종료일 입력 행은 날짜, 미입력 행은 `-` 표시
- [ ] CEO 지침·월별 매출·의사결정·핵심 메시지 정상
- [ ] 탭 스크롤 / `기존 폼으로 다운로드` 엑셀 / 빈 섹션 자동 숨김

> 데이터가 안 보이면 시트(J/K 컬럼 추가)와 Apps Script 재배포(7번)가 먼저 됐는지 확인하세요. `API_URL`은 기존 그대로면 수정 불필요.

---

## 6단계. 운영(Production)으로 반영 — Pull Request 병합

미리보기에서 문제가 없으면 main에 합칩니다.

1. GitHub 저장소 상단 **"Pull requests" 탭 > "New pull request"**
2. **base: `main`  ←  compare: `v5`** 로 설정 → **"Create pull request"** → **"Merge pull request"** → **"Confirm merge"**
3. main이 갱신되면서 Vercel이 **자동으로 Production 재배포** → 기존 운영 주소에 v5가 반영됩니다.
4. (선택) 병합 후 `v5` 브랜치는 **"Delete branch"** 로 정리해도 됩니다. (나중에 다시 만들 수 있음)

---

## 7단계. Apps Script(백엔드)는 별도 진행

Vercel/GitHub는 화면(프론트)만 담당합니다. 데이터 API는 구글 Apps Script이므로 따로 처리:
1. 구글시트 백업(사본 만들기) + 기존 `Code.gs` 텍스트 백업
2. 각 주차 시트 6번 섹션에 **J열=시작일, K열=종료일** 추가 (비워도 `-`로 표시)
3. `확장 프로그램 > Apps Script`에서 `Code.gs` 교체 → **배포 > 배포 관리 > 새 버전**으로 재배포 (웹앱 URL 유지 → `app.js`의 `API_URL` 수정 불필요)

상세 내용은 함께 드린 `배포가이드_v5.md`의 1·2·4단계를 참고하세요.

---

## 8. 롤백(되돌리기)

| 상황 | 방법 |
|---|---|
| 운영 반영 후 문제 발견 | **Vercel** 대시보드 > Deployments > 직전 정상 배포의 `⋯` > **"Promote to Production"** (= 즉시 롤백) |
| 코드 변경 자체를 취소 | **GitHub** 에서 해당 병합 커밋 **"Revert"** → 자동 재배포 |
| 백엔드 | Apps Script `버전 기록` 복원 또는 `배포 관리`에서 이전 버전 선택 |

---

## 9. 자주 막히는 포인트

- **브랜치 확인**: 3단계에서 반드시 화면 좌상단 브랜치가 `v5`인지 확인하고 커밋. 실수로 `main`에 바로 커밋하면 운영에 즉시 반영됩니다(이 경우 8번 롤백).
- **파일 위치**: v5 파일은 기존 파일과 **같은 폴더·같은 이름**으로 올려야 `index.html`이 `style.css`/`app.js`를 정상 참조합니다.
- **캐시**: 미리보기/운영에서 변경이 바로 안 보이면 브라우저 새로고침(Ctrl+F5).
- **빌드 실패 표시**: 정적 사이트라 보통 빌드 없이 Ready가 됩니다. 만약 Error가 뜨면 Vercel 프로젝트 Settings의 Framework Preset이 "Other"(또는 정적)로 돼 있는지 확인.
